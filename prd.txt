# PRD — Event Probability Intelligence Agent

## 项目名称

Event Probability Intelligence Agent（EPI Agent）

部署平台：

* OKX Onchain OS
* X Layer（后续）
* OKX Wallet 生态

---

# 1. 项目定义

一个基于 AI 的事件研究 Agent。

核心功能：

```text
监控现实世界事件
→ 映射相关预测市场
→ 分析事件导致的概率变化
→ 判断市场是否充分定价
```

Agent 输出：

* probability update
* market reaction
* pricing inefficiency
* research signal

核心研究对象：

```text
现实世界事件
如何改变 prediction market 的隐含概率。
```

# 4. 产品核心逻辑

当前产品逻辑升级为 market-first pricing engine。

核心抽象：

```text
Polymarket Market Universe
→ Market Benchmark Probability
→ Event / Macro Data
→ Fair Probability Estimate
→ Probability Delta
→ Mispricing / Repricing Signal
```

核心问题：

```text
某个 prediction market 的当前定价，
在已知事件、宏观数据、历史窗口和市场结构下，
是否偏离 EPI Agent 的 fair probability？
```

Agent 核心能力：

```text
识别：
underreaction
overreaction
delayed repricing
```

重要约束：

```text
Polymarket 当前价格是 benchmark，不是最终真概率。
EPI Agent 必须输出独立 fair probability，并解释它与 benchmark 的差异来源。
```

---

## 4.1 Market Benchmark 与初始偏差分析

系统接入 Polymarket 市场 API 后，第一步不是立即爬事件，而是先建立 market universe。

Market Universe 字段：

```text
market_id
condition_id
question
description
resolution_source
slug
event_slug
tags
outcomes
clob_token_ids
active
closed
end_date
volume
liquidity
updated_at
```

Market Benchmark 字段：

```text
best_bid
best_ask
midpoint
last_trade_price
outcome_price
spread
liquidity
volume_24h
volume_1w
price_timestamp
staleness_score
benchmark_confidence
```

Benchmark 优先级：

```text
best_bid / best_ask midpoint
→ outcomePrices
→ lastTradePrice
→ fallback market displayed probability
```

初始偏差分析：

```text
1. 检查同一 event 下互斥市场概率是否接近 100%
2. 检查 nested / threshold markets 是否满足单调性
3. 检查 wide spread、低 liquidity、stale price 导致的价格噪声
4. 检查同一宏观变量不同到期市场之间是否存在期限结构异常
5. 检查相关市场之间是否违反基本条件概率关系
```

示例：

```text
P(CPI > 3%) >= P(CPI > 4%) >= P(CPI > 5%)
P(Fed cuts at least 1 time) >= P(Fed cuts at least 2 times)
P(candidate wins nomination) >= P(candidate wins general election | must be nominee first 的近似约束)
```

## 4.2 自有定价策略

EPI Agent 需要为每类市场维护定价策略模板。

第一阶段使用可解释的 event-driven repricing model，而不是黑箱交易模型。

最小可行公式：

```text
prior = market_benchmark_probability
event_delta = direction
            × surprise_score
            × market_sensitivity
            × time_decay
            × source_reliability
            × liquidity_adjustment

fair_probability = clamp(prior + event_delta, 0.01, 0.99)
```

更稳健版本：

```text
logit(p) = ln(p / (1 - p))
posterior_logit = logit(prior) + evidence_weight
fair_probability = sigmoid(posterior_logit)
```

输出：

```text
benchmark_probability
fair_probability
model_delta
market_delta
repricing_gap
confidence_score
signal_strength
```

偏差判断：

```text
repricing_gap > threshold
→ underreaction

repricing_gap < -threshold
→ overreaction

abs(repricing_gap) <= threshold
→ repriced appropriately
```

---

## 4.3 历史窗口、重大事件与去重抓取

每个市场需要一个有效历史窗口。

默认：

```text
lookback_window = 180 days
```

可调整：

```text
Fed / inflation near-term markets: 90-180 days
annual macro threshold markets: 365 days
election markets: 180-730 days
geopolitical conflict markets: 30-180 days
company valuation / M&A markets: 180-365 days
```

数据抓取策略：

```text
首次同步：
拉取 lookback_window 内的市场价格、宏观数据、重大事件。

增量同步：
只拉取 last_successful_sync_at 之后的新数据。

重大数据发布：
按 release calendar 触发即时更新定价模型。

修正数据：
保留 vintage，不覆盖历史可见值。
```

去重键：

```text
source_name
series_id / market_id / event_id
observation_time
release_time
content_hash
revision_version
```

